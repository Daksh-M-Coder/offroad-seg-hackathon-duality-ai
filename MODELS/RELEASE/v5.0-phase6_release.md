## 🚀 Release Notes

This is the **ultimate checkpoint** of the repository, representing a massive **86.0% overall improvement** from the Phase 1 baseline. It achieves a dominant `0.5527` score utilizing a three-pronged strategy targeting the boundaries between visually similar desert artifacts (Rocks vs Ground Clutter vs Landscape).

### Key Improvements in this Release:
* **Boundary Loss**: Added an active `BoundaryLoss` module (weighted at `0.5`) combining distance maps and boundary masks to explicitly penalize ragged edges. Total Loss = `1.0*Focal + 0.5*Dice + 0.5*Boundary`.
* **Deepened Unfreezing**: Unfroze Block 9 in the DINOv2 backbone (now blocks 9, 10, 11) to give the boundary features more representational capacity deep in the transformer.
* **Multi-Scale TTA Evaluation**: Replaced simple HFlip TTA with an **8-pass Multi-Scale TTA** protocol during evaluation, testing images at `[0.9x, 1.0x, 1.1x, 1.2x]` scales across normal and flipped orientations, yielding an instant `+0.016` IoU boost.

### Architecture Details
* **Backbone**: `facebook/dinov2-base` (768-dim) — **Blocks 0-8 Frozen, Blocks 9-11 Unfrozen**
* **Segmentation Head**: `UPerNetHead`
* **Input Resolution**: `644 × 364`

### Note on Checkpoint File Size
**File size increased to 282MB**. The optimizer state dictionary expanded again to accommodate the parameters for Block 9, plus tracking metrics for the new Boundary Loss tracking state.

### How to Load
For inference, only initialize the base backbone architecture, then insert the updated blocks using the checkpoint:
```python
# Override blocks 9-11
current_backbone_state = model.backbone.state_dict()
current_backbone_state.update(checkpoint['backbone_state_dict'])
model.backbone.load_state_dict(current_backbone_state)
```
