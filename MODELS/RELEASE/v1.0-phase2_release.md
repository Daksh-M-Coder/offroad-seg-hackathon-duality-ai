## 🚀 Release Notes

This checkpoint marks the end of Phase 2, establishing a heavily optimized baseline before architectural changes. Compared to the provided Phase 1 baseline (`0.2971`), this model achieves a **35.8% relative improvement** using the same DINOv2 `vits14` backbone and ConvNeXt head.

### Key Improvements in this Release:
* **Weighted CrossEntropy Loss**: Inverse pixel frequency weighting added to counter the severe class imbalance (Sky=34%, Logs=0.07%).
* **Optimization Upgrade**: Switched from SGD to **AdamW** (`lr=5e-4`, `weight_decay=1e-4`).
* **Learning Rate Schedule**: Implemented `CosineAnnealingLR` for smooth convergence.
* **Data Augmentation**: Integrated comprehensive `albumentations` pipeline including `HorizontalFlip`, `VerticalFlip`, `ShiftScaleRotate`, `GaussianBlur`, `RandomBrightnessContrast`, and `HueSaturationValue`.
* **Mixed Precision (AMP)**: Enabled FP16 training, effectively halving memory usage while maintaining accuracy.

### Architecture Details
* **Backbone**: `facebook/dinov2-small` (384-dim embeddings) — **Fully Frozen**
* **Segmentation Head**: Custom `ConvNeXtHead` (Conv2d → BatchNorm2d → ReLU)
* **Input Resolution**: `476 × 266`

This checkpoint contains the complete Python dictionary tracking `model_state_dict`, `optimizer_state_dict`, and `config`. It operates purely on 384-dimensional features.
