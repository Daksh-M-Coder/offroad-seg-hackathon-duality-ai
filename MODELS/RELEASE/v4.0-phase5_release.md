## 🚀 Release Notes

This release breaks through the Phase 4 frozen-backbone barrier (`0.5169`) by introducing **Controlled Backbone Fine-Tuning**. By carefully unfreezing the deepest layers of DINOv2, the model adapts its general pre-trained features specifically to the semantics of the synthetic desert dataset.

### Key Improvements in this Release:
* **Partial Unfreezing**: Blocks 10 and 11 of the DINOv2 backbone are now unfrozen and updated during training.
* **Differential Learning Rates**: Implemented a two-tier LR strategy to prevent catastrophic forgetting:
  * Backbone blocks 10-11 LR: `5e-6`
  * UPerNet head LR: `2e-4` (40× higher)
* **Gradient Clipping**: Added `max_norm=1.0` clipping to specifically protect the unfrozen ViT blocks from destabilizing spike gradients.
* **Overfit Monitoring**: Introduced a custom Train-Val gap tracker, ensuring the fine-tuning did not memorize the training set (max gap reached was only `0.038`).

### Architecture Details
* **Backbone**: `facebook/dinov2-base` (768-dim) — **Blocks 0-9 Frozen, Blocks 10-11 Unfrozen**
* **Segmentation Head**: `UPerNetHead`
* **Loss**: Focal + Dice (`1.0`, `0.5`)

### Note on Checkpoint File Size
**File size jump from 39MB to 201MB**. Since blocks 10 and 11 are now unfrozen, the AdamW optimizer state dictionary tracks momentum and variance for millions of new parameters inside the backbone blocks.
