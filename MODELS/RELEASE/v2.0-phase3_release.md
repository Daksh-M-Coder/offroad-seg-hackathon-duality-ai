## 🚀 Release Notes

This checkpoint marks a massive architectural leap in our segmentation pipeline, achieving a **+27.9% relative IoU gain** over Phase 2 and crossing the critical 0.50 threshold. This release completely replaces the segmentation head and upgrades the backbone.

### Key Improvements in this Release:
* **Backbone Upgrade**: Upgraded from ViT-Small (384-dim) to **ViT-Base (768-dim)**, doubling the feature density.
* **Head Replacement**: Replaced the simple ConvNeXt head with a full **UPerNet architecture**. This brings Multi-Scale Feature Pyramid Network (FPN) and Pyramid Pooling Module (PPM) capabilities.
* **Loss Function Overhaul**: Switched from Weighted CrossEntropy to a combined **Focal Loss + Dice Loss** (1:1 ratio), prioritizing hard-to-classify pixels and structure overlap.
* **Resolution Bump**: Increased training resolution from 476×266 to **644×364** (+36% more pixels) for better small-object detection (logs, rocks).

### Architecture Details
* **Backbone**: `facebook/dinov2-base` (768-dim embeddings, patch size 14) — **Fully Frozen**
* **Segmentation Head**: `UPerNetHead` (In channels: `[768, 768, 768, 768]`, channels: `512`)
* **Input Resolution**: `644 × 364`
* **Optimizer**: AdamW (`lr=4e-4`)

### Usage
Requires loading both the `facebook/dinov2-base` backbone and initializing the UPerNet head before applying the `checkpoint['model_state_dict']`.
